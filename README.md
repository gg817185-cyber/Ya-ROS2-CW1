Данный пакет представляет собой систему управления учебным роботом, которая включает в себя мониторинг заряда батареи, симуляцию датчика расстояния, агрегатор статуса и логический контроллер движения. Пакет также содержит URDF-модель робота.

1. Подготовка и сборка пакета
Перед запуском системы необходимо убедиться, что пакет корректно собран и все зависимости установлены.

Команды для сборки:

Bash

# Переход в корень воркспейса
cd ~/ros2_ws

# Исправление предупреждения о локали (если требуется)
export LC_ALL=C.UTF-8

# Сборка конкретного пакета
colcon build --packages-select exam_robot --symlink-install

# Активация окружения
source install/setup.bash
Критерий успеха: Сообщение "Summary: 1 package finished" без ошибок сборки.

2. Запуск системы
Вся система (все 4 узла и модель робота) запускается одной командой с помощью launch-файла.

Команда запуска:

Bash

ros2 launch exam_robot robot_system.launch.py
Запускаемые узлы:

battery_node: Симулирует разряд батареи.
distance_sensor: Рассчитывает расстояние до препятствия.
status_display: Определяет общее состояние робота.
robot_controller: Формирует команды скорости /cmd_vel.
robot_state_publisher: Публикует структуру робота (TF) из URDF.
3. Проверка взаимодействия (Топики)
Для проверки необходимо открыть новый терминал, выполнить в нем source install/setup.bash и запустить диагностические команды.

Список активных топиков:
Bash

ros2 topic list
В списке должны присутствовать: /battery_level, /distance, /robot_status, /cmd_vel, /tf, /robot_description.

Проверка частоты публикации (Hz):
Согласно техническому заданию, узлы должны работать со следующей частотой:

Батарея (1 Hz):
ros2 topic hz /battery_level
Датчик расстояния (5 Hz):
ros2 topic hz /distance
Статус (2 Hz):
ros2 topic hz /robot_status
Контроллер (10 Hz):
ros2 topic hz /cmd_vel
4. Проверка структуры робота (TF дерево)
Для подтверждения корректной загрузки URDF-модели и работы транскуставов (joints), необходимо проверить дерево TF.

Команда для генерации дерева:

Bash

ros2 run tf2_tools view_frames
Ожидаемый результат:
Будет создан файл frames.pdf. Дерево трансформаций должно быть единым и содержать 4 обязательных звена:

base_link (центр робота)
left_wheel (левое колесо)
right_wheel (правое колесо)
sensor_link (датчик расстояния)
5. Логика поведения системы (Демонстрация)
В процессе работы в консоли launch-файла можно наблюдать автоматическую смену режимов:

Норма (ALL OK): Робот движется со скоростью 0.3 м/с, расстояние уменьшается.
Препятствие (WARNING): При расстоянии < 1.0м робот начинает вращаться на месте.
Низкий заряд (WARNING): При заряде < 20% скорость падает до 0.1 м/с.
Критическая остановка (CRITICAL): При заряде < 10% или дистанции < 0.7м робот полностью останавливается (0.0 м/с).
